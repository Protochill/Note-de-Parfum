# Note de Parfum — Каталог парфюмерии

## 1. Тема и концепция

**Название приложения:** Note de Parfum

**Краткое описание:**
«Note de Parfum» — это динамический веб-каталог для ценителей парфюмерии. Приложение позволяет пользователям изучать детальную информацию об ароматах, включая пирамиду нот, парфюмеров и историю создания. Зарегистрированные пользователи могут формировать личную коллекцию «Избранное», оставлять отзывы и получать персональные рекомендации на основе своих предпочтений. Проект реализован с упором на простоту, скорость загрузки и удобство использования.

**Целевая аудитория:**
* Люди, интересующиеся парфюмерией (от новичков до нишевых коллекционеров).
* Покупатели, которые хотят подробно изучить аромат перед покупкой в интернет-магазине.
* Пользователи, ведущие цифровой дневник своих парфюмерных открытий.

**Ключевой функционал:**
* Просмотр каталога парфюмерии с фильтрацией по брендам, нотам, годам выпуска и половой принадлежности.
* Детальные карточки товаров с описанием пирамиды нот, характеристиками и отзывами.
* Регистрация, аутентификация и восстановление пароля.
* Личный кабинет пользователя с возможностью добавлять ароматы в избранное и писать отзывы.
* Поиск по названию аромата или бренду.
* Административная панель для управления каталогом (добавление/редактирование брендов, ароматов, нот).

## 3. Технологический стек и его обоснование

Для реализации проекта **Note de Parfum** был выбран прагматичный технологический стек, ориентированный на простоту, скорость разработки и достаточную производительность для компактного парфюмерного каталога. Отказ от тяжелых фреймворков в пользу проверенных и легковесных решений позволяет сосредоточиться на бизнес-логике и быстром запуске.

### Frontend (Клиентская часть)

*   **Язык и архитектура: Чистый JavaScript (Vanilla JS) + HTML + CSS**
    *   **Обоснование:** Использование чистого JavaScript без фреймворков (таких как React, Vue или Angular) является осознанным решением, продиктованным масштабом проекта. Для компактного каталога парфюмерии Note de Parfum внедрение тяжеловесных библиотек создало бы избыточную сложность. Такой подход дает несколько преимуществ:
        1.  **Отсутствие лишних абстракций:** Разработка ведется напрямую с DOM-деревом, что обеспечивает прямой контроль над элементами интерфейса — динамической фильтрацией ароматов, формой поиска по нотам, личным кабинетом с избранным. Это упрощает реализацию специфической динамики.
        2.  **Легковесность и производительность:** Финальный код не содержит кода фреймворка, что положительно сказывается на скорости загрузки страниц и работе приложения на любых устройствах, что особенно важно для мобильных пользователей, изучающих ароматы "на ходу".
        3.  **Простота поддержки:** HTML, CSS и "ванильный" JavaScript являются стандартами веба. Любой разработчик, знакомый с основами фронтенда, сможет быстро понять структуру проекта и приступить к его доработке (например, добавлению нового блока с пирамидой нот) без необходимости изучения специфики конкретного фреймворка.

*   **Обмен данными с сервером: Fetch API**
    *   **Обоснование:** Для асинхронного взаимодействия с бэкендом используется встроенный в браузер Fetch API. Это современный стандарт, который позволяет отправлять HTTP-запросы и обрабатывать ответы без перезагрузки страницы. Благодаря Fetch API, при применении фильтров в каталоге или добавлении аромата в избранное, страница не перезагружается, создавая плавный пользовательский опыт. Выбор Fetch API вместо библиотек вроде Axios или jQuery обусловлен отсутствием необходимости подключать дополнительные зависимости, что сохраняет проект легким и соответствует общей концепции минимализма.

### Backend (Серверная часть)

*   **Среда выполнения: Node.js**
    *   **Обоснование:** Node.js обеспечивает высокую производительность серверной части благодаря своей неблокирующей, событийно-ориентированной архитектуре. Это идеально подходит для обработки множества одновременных запросов от пользователей к API каталога с ароматами, особенно при пиковых нагрузках в период праздников. Ключевым преимуществом является использование единого языка — JavaScript — как на клиенте, так и на сервере. Это унифицирует процессы разработки, упрощает поиск и исправление ошибок и снижает порог входа для новых разработчиков в проект.

*   **Фреймворк: Express.js**
    *   **Обоснование:** Express — это минималистичный и гибкий веб-фреймворк для Node.js, который стал стандартом в индустрии. Он предоставляет удобные инструменты для маршрутизации (обработки запросов к `/api/perfumes`, `/api/brands`, `/auth/login`, `/profile`), интеграции с middleware и обработки ошибок. Express не навязывает жесткой структуры проекта, что позволяет быстро создать легкое и эффективное REST API именно под нужды Note de Parfum, не перегружая его избыточным функционалом.

*   **База данных: MySQL**
    *   **Обоснование:** Для хранения данных парфюмерного каталога оптимально подходит реляционная база данных MySQL. Структура магазина оперирует четкими и взаимосвязанными сущностями: «Пользователи», «Ароматы», «Бренды», «Ноты», «Отзывы». MySQL идеально справляется с задачей поддержания целостности этих связей (например, связь аромата с конкретным брендом и множеством нот) и обеспечения надежности хранения. Кроме того, MySQL является проверенным и широко поддерживаемым решением, что гарантирует простоту развертывания на большинстве хостинговых платформ.

*   **Безопасность и сессии: bcrypt и express-session**
    *   **Обоснование:**
        1.  **`bcrypt` для хеширования паролей:** Безопасность пользовательских данных является приоритетом. `bcrypt` используется для надежного хеширования паролей перед сохранением в базу данных. В отличие от простых хеш-функций (например, MD5), `bcrypt` автоматически генерирует уникальную «соль» для каждого пароля, что делает бесполезными атаки с использованием радужных таблиц. Кроме того, алгоритм является намеренно медленным, что значительно затрудняет злоумышленникам подбор паролей методом перебора (брутфорс).
        2.  **`express-session` для авторизации:** Для управления состоянием авторизации пользователя используется middleware `express-session`. Он позволяет сохранять данные о пользователе (например, его идентификатор и роль) между различными HTTP-запросами. Такой подход безопасен, так как на клиенте (в браузере) хранится только уникальный идентификатор сессии, а не конфиденциальные данные. Это позволяет пользователям оставаться авторизованными при переходе между страницами каталога и личным кабинетом.

### Инфраструктура (Хостинг и деплой)

*   **Frontend хостинг:** **GitHub Pages** или **Vercel**
    *   **Обоснование:** Обе платформы идеально подходят для хостинга статических сайтов, созданных с помощью HTML, CSS и JavaScript. Они предлагают простую интеграцию с Git-репозиторием (автоматический деплой при пуше в основную ветку), бесплатные SSL-сертификаты и быструю доставку контента до пользователей по всему миру благодаря встроенной CDN (Content Delivery Network).

*   **Backend хостинг и База данных:** **Render** / **Railway** или **VPS**
    *   **Обоснование:** Для размещения серверной части на Node.js и базы данных MySQL оптимальным выбором станут платформы, предоставляющие управляемую инфраструктуру (PaaS), такие как Render или Railway. Они позволяют развернуть приложение и базу данных в несколько кликов, автоматически управляя окружением, SSL и масштабированием. Это избавляет от необходимости администрировать виртуальный сервер (VPS) на начальных этапах и позволяет сфокусироваться исключительно на разработке функционала Note de Parfum.

## 4. Визуальное проектирование интерфейса


**Страница:** Каталог (`/catalog`)
*   **Тип доступа:** публичный
*   **Основные блоки:**
    1.  **Шапка:** Логотип "Note de Parfum", навигация (Главная, Каталог, Бренды), поле поиска, иконки профиля/входа и избранного.
    2.  **Сайдбар (левый):** Блоки фильтрации: "Бренды" (список с чекбоксами), "Ноты" (поле поиска по нотам + чекбоксы популярных), "Пол" (радиокнопки), "Год выпуска" (ползунок диапазона).
    3.  **Основная область (центр):** Заголовок "Каталог ароматов", информация о количестве найденных товаров, элементы управления сортировкой (по названию, по популярности, по новизне). Сетка из карточек ароматов.
    4.  **Карточка аромата:** Изображение флакона, название бренда, название аромата, год выпуска, краткий состав (верхние ноты), средняя оценка, кнопка "В избранное" (сердечко).
    5.  **Футер:** Контакты, ссылки на соцсети, информация о проекте.
*   **Динамический контент:** Все карточки и фильтры. Данные загружаются с API (`GET /api/perfumes` с query-параметрами для фильтрации). Количество товаров обновляется динамически при применении фильтров.

**Страница:** Детальная страница аромата (`/catalog/:id`)
*   **Тип доступа:** публичный
*   **Основные блоки:**
    1.  **Шапка и Футер:** Аналогичны главной странице.
    2.  **Основная область:** Две колонки. Левая: крупное изображение флакона. Правая: название бренда и аромата, кнопка "В избранное", средняя оценка и количество отзывов.
    3.  **Блок "Пирамида нот":** Визуальное представление (например, три колонки) с разделением на верхние, средние (сердце) и базовые ноты. Каждая нота — это кликабельная ссылка для поиска всех ароматов с этой нотой.
    4.  **Блок "О аромате":** Описание, год создания, имя парфюмера.
    5.  **Блок "Отзывы":** Список отзывов пользователей с оценками и текстом. Кнопка "Написать отзыв" (доступна только авторизованным).
    6.  **Блок "Вам также может понравиться":** Горизонтальный слайдер с 4-5 карточками ароматов (рекомендации на основе текущего).
*   **Динамический контент:** Вся информация об аромате (`GET /api/perfumes/:id`), список отзывов (`GET /api/perfumes/:id/reviews`), блок рекомендаций (`GET /api/perfumes/:id/recommendations`).

## 5. Схема данных и API

### Модель User (таблица: users)

| Поле | Тип | Описание |
|:---|:---|:---|
| id | INT, PK, AUTO_INCREMENT | Уникальный идентификатор пользователя |
| login | VARCHAR, UNIQUE | Логин пользователя (уникальный) |
| password | VARCHAR | Хешированный пароль (bcrypt) |
| phone | VARCHAR, UNIQUE | Номер телефона пользователя (уникальный) |
| role | ENUM('user', 'admin') | Роль пользователя (user или admin) |
| favorites | JSON | Массив ID избранных ароматов (например, [1, 5, 12]) |
| created_at | TIMESTAMP | Дата и время регистрации |
| updated_at | TIMESTAMP | Дата и время последнего обновления |

**Особенности модели:**
- Пароль хранится **только в виде bcrypt-хеша**, никогда не сохраняется в открытом виде
- Логин и телефон проходят проверку на уникальность при регистрации и обновлении профиля
- Роль определяет доступ к админ-панели
- Избранное хранится в JSON прямо в таблице пользователя — это избавляет от отдельной таблицы favorites и упрощает запросы

---

### Модель Perfume (таблица: perfumes) — основная сущность

| Поле | Тип | Описание |
|:---|:---|:---|
| id | INT, PK, AUTO_INCREMENT | Уникальный идентификатор аромата |
| name | VARCHAR | Название аромата |
| brand | VARCHAR | Название бренда |
| brand_country | VARCHAR | Страна бренда |
| brand_description | TEXT | Описание бренда |
| brand_logo_url | VARCHAR | Ссылка на логотип бренда |
| brand_founded_year | INT | Год основания бренда |
| perfumer | VARCHAR | Имя парфюмера, создавшего аромат |
| release_year | INT | Год выпуска аромата |
| gender | ENUM('male', 'female', 'unisex') | Пол, для которого предназначен аромат |
| description | TEXT | Подробное описание аромата |
| image_url | VARCHAR | Ссылка на изображение флакона |
| top_notes | JSON | Массив верхних нот (например, ["бергамот", "лимон"]) |
| middle_notes | JSON | Массив нот сердца (например, ["жасмин", "роза"]) |
| base_notes | JSON | Массив базовых нот (например, ["ваниль", "мускус"]) |
| average_rating | DECIMAL(2,1) | Средняя оценка на основе отзывов |
| created_at | TIMESTAMP | Дата и время добавления в каталог |
| updated_at | TIMESTAMP | Дата и время последнего обновления |

**Особенности модели:**
- Информация о бренде хранится прямо в таблице ароматов — это избавляет от отдельной таблицы brands и необходимости делать JOIN
- Если у бренда несколько ароматов, данные бренда дублируются (для масштаба проекта это допустимо)
- Ноты хранятся в JSON-формате, что позволяет хранить переменное количество нот без изменения структуры таблицы
- Поле average_rating обновляется автоматически при добавлении, изменении или удалении отзывов

---

### Модель Review (таблица: reviews)

| Поле | Тип | Описание |
|:---|:---|:---|
| id | INT, PK, AUTO_INCREMENT | Уникальный идентификатор отзыва |
| user_id | INT, FOREIGN KEY | ID пользователя (связь с таблицей users) |
| perfume_id | INT, FOREIGN KEY | ID аромата (связь с таблицей perfumes) |
| user_login | VARCHAR | Логин пользователя (денормализация для быстрого отображения) |
| rating | TINYINT | Оценка от 1 до 5 |
| text | TEXT | Текст отзыва |
| created_at | TIMESTAMP | Дата и время написания отзыва |
| updated_at | TIMESTAMP | Дата и время последнего редактирования |

**Особенности модели:**
- Составной уникальный ключ (user_id, perfume_id) гарантирует, что пользователь может оставить только один отзыв на конкретный аромат
- Денормализация: поле user_login хранится прямо в отзыве — это позволяет при загрузке отзывов не делать дополнительный JOIN к таблице пользователей
- Если пользователь меняет логин, необходимо обновить все его отзывы (можно автоматизировать триггером)
- При добавлении, обновлении или удалении отзыва автоматически пересчитывается average_rating в таблице perfumes

  
##  API Эндпоинты

### Аутентификация
| Метод | Эндпоинт | Описание | Тело запроса | Ответ |
|-------|----------|----------|--------------|--------|
| POST | `/api/auth/register` | Регистрация нового пользователя | `{ fullName, email, password, phone }` | `{ success, userId }` |
| POST | `/api/auth/login` | Вход в систему | `{ email, password }` | `{ token, user }` |
| POST | `/api/auth/logout` | Выход из системы | - | `{ success }` |

### Товары (основная сущность)
| Метод | Эндпоинт | Описание | Параметры | Ответ |
|-------|----------|----------|-----------|--------|
| GET | `/api/products` | Получение списка товаров | `page`, `limit`, `brand`, `family`, `priceFrom`, `priceTo` | Массив товаров |
| GET | `/api/products/:id` | Получение детальной информации о товаре | `id` в URL | Полный объект товара |
| GET | `/api/products/search` | Поиск товаров | `q` (поисковый запрос) | Массив найденных товаров |
