# Note de Parfum — Каталог парфюмерии

**Название приложения:** Note de Parfum

**Краткое описание:**
«Note de Parfum» — это динамический веб-каталог для ценителей парфюмерии. Приложение позволяет пользователям изучать детальную информацию об ароматах, включая пирамиду нот, парфюмеров и историю создания. Зарегистрированные пользователи могут формировать личную коллекцию «Избранное», оставлять отзывы и получать персональные рекомендации на основе своих предпочтений. Проект реализован с упором на простоту, скорость загрузки и удобство использования.

**Целевая аудитория:**
- Люди, интересующиеся парфюмерией (от новичков до нишевых коллекционеров)
- Покупатели, которые хотят подробно изучить аромат перед покупкой в интернет-магазине
- Пользователи, ведущие цифровой дневник своих парфюмерных открытий

**Ключевой функционал:**
- Просмотр каталога парфюмерии с фильтрацией по брендам, нотам, годам выпуска и половой принадлежности
- Детальные карточки товаров с описанием пирамиды нот, характеристиками и отзывами
- Регистрация, аутентификация и восстановление пароля
- Личный кабинет пользователя с возможностью добавлять ароматы в избранное и писать отзывы
- Поиск по названию аромата или бренду
- Административная панель для управления каталогом (добавление/редактирование ароматов)

## Технологический стек и его обоснование

### Frontend (Клиентская часть)

**Язык и архитектура: Чистый JavaScript (Vanilla JS) + HTML + CSS**

Использование чистого JavaScript без фреймворков является осознанным решением, продиктованным масштабом проекта. Для компактного каталога парфюмерии внедрение тяжеловесных библиотек создало бы избыточную сложность.

**Преимущества подхода:**
- **Отсутствие лишних абстракций** — разработка ведется напрямую с DOM-деревом, обеспечивая полный контроль над интерфейсом
- **Легковесность и производительность** — финальный код не содержит кода фреймворка, что ускоряет загрузку страниц
- **Простота поддержки** — любой разработчик быстро разберется в структуре проекта

**Обмен данными с сервером: Fetch API**

Для асинхронного взаимодействия с бэкендом используется встроенный Fetch API. Это современный стандарт, позволяющий отправлять запросы без перезагрузки страницы. Отказ от библиотек вроде Axios обусловлен отсутствием необходимости в дополнительных зависимостях.

### Backend (Серверная часть)

**Среда выполнения: Node.js**

Node.js обеспечивает высокую производительность благодаря неблокирующей, событийно-ориентированной архитектуре. Ключевое преимущество — использование единого языка (JavaScript) на клиенте и сервере, что упрощает разработку и поддержку.

**Фреймворк: Express.js**

Express — минималистичный веб-фреймворк, предоставляющий удобные инструменты для маршрутизации и обработки запросов. Он не навязывает жесткой структуры, позволяя быстро создать легкое REST API под нужды проекта.

**База данных: MySQL**

MySQL оптимально подходит для хранения четких взаимосвязанных сущностей: пользователи, ароматы, бренды, отзывы. Это проверенное решение с высокой надежностью и простотой развертывания.

**Безопасность и сессии: bcrypt и express-session**

- **bcrypt** — обеспечивает надежное хеширование паролей с автоматической генерацией "соли", защищая от атак с использованием радужных таблиц
- **express-session** — управляет состоянием авторизации, храня на клиенте только уникальный идентификатор сессии

### Инфраструктура (Хостинг и деплой)

- **Frontend хостинг:** GitHub Pages / Vercel — автоматический деплой при пуше в репозиторий, бесплатные SSL-сертификаты, встроенная CDN
- **Backend хостинг и БД:** Render / Railway — управляемая инфраструктура, автоматическое управление окружением и масштабированием

## Схема данных и API

### Модель User (таблица: users)

| Поле | Тип | Описание |
|:---|:---|:---|
| id | INT, PK, AUTO_INCREMENT | Уникальный идентификатор пользователя |
| login | VARCHAR, UNIQUE | Логин пользователя (уникальный) |
| password | VARCHAR | Хешированный пароль (bcrypt) |
| phone | VARCHAR, UNIQUE | Номер телефона (уникальный) |
| role | ENUM('user', 'admin') | Роль пользователя |
| favorites | JSON | Массив ID избранных ароматов |
| created_at | TIMESTAMP | Дата регистрации |
| updated_at | TIMESTAMP | Дата последнего обновления |

**Особенности модели:**
- Пароль хранится только в виде bcrypt-хеша
- Логин и телефон уникальны
- Роль определяет доступ к админ-панели
- Избранное хранится в JSON (без отдельной таблицы)

---

### Модель Perfume (таблица: perfumes)

| Поле | Тип | Описание |
|:---|:---|:---|
| id | INT, PK, AUTO_INCREMENT | Уникальный идентификатор аромата |
| name | VARCHAR | Название аромата |
| brand | VARCHAR | Название бренда |
| brand_country | VARCHAR | Страна бренда |
| brand_description | TEXT | Описание бренда |
| brand_logo_url | VARCHAR | Логотип бренда |
| brand_founded_year | INT | Год основания |
| release_year | INT | Год выпуска аромата |
| gender | ENUM('male', 'female', 'unisex') | Половая принадлежность |
| description | TEXT | Описание аромата |
| image_url | VARCHAR | Изображение флакона |
| top_notes | JSON | Массив верхних нот |
| middle_notes | JSON | Массив нот сердца |
| base_notes | JSON | Массив базовых нот |
| average_rating | DECIMAL(2,1) | Средняя оценка |
| created_at | TIMESTAMP | Дата добавления |
| updated_at | TIMESTAMP | Дата обновления |

**Особенности модели:**
- Информация о бренде хранится прямо в таблице ароматов (без отдельной таблицы brands)
- Допустимо дублирование данных бренда для разных ароматов
- Ноты хранятся в JSON для гибкости
- average_rating обновляется автоматически при изменении отзывов

---

### Модель Review (таблица: reviews)

| Поле | Тип | Описание |
|:---|:---|:---|
| id | INT, PK, AUTO_INCREMENT | Уникальный идентификатор отзыва |
| user_id | INT, FOREIGN KEY | ID пользователя |
| perfume_id | INT, FOREIGN KEY | ID аромата |
| user_login | VARCHAR | Логин пользователя (денормализация) |
| rating | TINYINT | Оценка (1-5) |
| text | TEXT | Текст отзыва |
| created_at | TIMESTAMP | Дата написания |
| updated_at | TIMESTAMP | Дата редактирования |

**Особенности модели:**
- Составной уникальный ключ (user_id, perfume_id) — один отзыв на аромат от пользователя
- Денормализация user_login для быстрого отображения без JOIN
- При изменении логина пользователя обновляются все его отзывы
